## 一、用户画像

### 1.1 登机口服务人员（核心用户）

**特征**：
- 年龄：25-45岁
- 工作经验：1-10年
- 操作环境：机场登机口，噪音环境（70-85dB）
- 工作节奏：快节奏，多任务并行

**需求**：
- 快速完成广播，减少查找时间
- 避免口误，保证播报准确
- 支持多语种播报（无需外包）
- 操作简单，无需培训即可上手

**痛点**：
- 纸质手册查找慢（2-5分钟）
- 人工播报容易出错
- 多语种需要多人协作或外包
- 紧急情况响应慢

**使用频率**：每日20-50次广播

---

**特征**：
- 年龄：30-50岁
- 管理范围：5-10个登机口

**需求**：
- 监控广播服务质量

- 管理广播词内容
- 培训和督导服务人员

**使用频率**：每周2-3次查看报表

**特征**：
- 年龄：28-40岁
- 职责：系统维护、配置管理

## 三、使用场景

### 3.1 场景一：正常登机广播

**触发条件**：航班开始登机

**操作流程**：
1. 打开应用（0.5秒）
2. 点击"正常登机"场景卡片（1秒）
3. 确认参数（航班号、登机口、目的地已自动填充）（2秒）
4. 选择播报语言：中文、英语（2秒）
5. 点击播放按钮（0.5秒）
6. 音频生成并播放（2秒）

**总耗时**：约8秒

**关键路径**：场景选择 → 参数确认 → 播放

**错误处理**：
- 参数缺失：红框提示 + 必填标记
- 音频生成失败：自动重试 + 错误提示
- 设备连接失败：提示检查蓝牙 + 本地降级

---

### 3.2 场景二：登机催促广播

**触发条件**：登机结束前10分钟

**操作流程**：
1. 打开应用（0.5秒）
2. 点击"登机催促"场景卡片（1秒）
3. 确认参数（2秒）
4. 点击播放按钮（0.5秒）
5. 音频生成并播放（2秒）

**总耗时**：约6秒

**快捷方式**：从历史记录快速重播（3秒）

---

### 3.3 场景三：航班延误通知

**触发条件**：航班延误发布

**操作流程**：
1. 打开应用（0.5秒）
2. 点击"延误通知"场景卡片（1秒）
3. 填写延误时间、预计起飞时间（5秒）
4. 选择播报语言（2秒）
5. 预览播报内容（1秒）
6. 点击播放按钮（0.5秒）
7. 音频生成并播放（2秒）

**总耗时**：约12秒

**特点**：需要填写额外参数，支持预览

---

### 3.4 场景四：登机口变更通知

**触发条件**：登机口变更

**操作流程**：
1. 打开应用（0.5秒）
2. 点击"登机口变更"场景卡片（1秒）
3. 填写原登机口、新登机口（5秒）
4. 确认播报语言（2秒）
5. 点击播放按钮（0.5秒）
6. 音频生成并播放（2秒）

**总耗时**：约11秒

**关键点**：信息准确度要求高，支持二次确认

---

### 3.5 场景五：紧急情况广播

**触发条件**：突发安全事件

**操作流程**：
1. 打开应用（0.5秒）
2. 点击红色"紧急广播"按钮（1秒）
3. 选择紧急类型（疏散/撤离/其他）（1秒）
4. 二次确认弹窗（1秒）
5. 点击确认播放（0.5秒）
6. 音频立即播放（1秒）

**总耗时**：约5秒

**特点**：
- 一键触发，最小化操作
- 高优先级，打断当前任务
- 支持物理音量键紧急停止
- 本地优先，不依赖网络

## 五、技术选型建议

### 5.1 移动端TTS方案

**推荐方案：离线为主，云端为辅**

| 方案 | 优先级 | 适用场景 | 优势 |
|------|--------|----------|------|
| **讯飞离线TTS SDK** | P0 | 日常广播 | 语音质量高、离线稳定、多语种丰富 |
| **iOS/Android系统TTS** | 备用 | 降级方案 | 免费原生支持、无依赖 |
| **阿里云NUI云端TTS** | P2 | 高级需求 | 语音质量最佳、持续更新 |
| **F5-TTS自研** | 可选 | 长期方案 | 控制力强、无第三方依赖 |

**技术实现**：

**iOS**：
```
主要：讯飞iOS离线TTS SDK
备用：AVFoundation + 本地语音包
API：AVAudioSession（音频路由管理）
```

**Android**：
```
主要：讯飞Android离线TTS SDK
备用：Android TextToSpeech API
API：AudioManager（音频焦点管理）
```

**语言包管理**：
- 按需下载（首次使用时）
- 支持批量下载（WiFi环境）
- 每个语言包大小：50-100MB
- 定期更新提示

### 5.4 音频输出方案

**方案1：蓝牙连接（MVP）**
- 优点：无线、灵活、兼容性好
- 缺点：延迟100-300ms
- 实施难度：低

**方案2：有线连接（v1.5）**
- 优点：无延迟、稳定、音质好
- 缺点：需要适配器
- 实施难度：中

**方案3：PA系统集成（v2.0）**
- 优点：音质最佳、覆盖广
- 缺点：成本高、需要机场支持
- 实施难度：高

**推荐路线**：蓝牙 → 有线 → PA系统
