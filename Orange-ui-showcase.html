Color with purpose, not decoration

有目的的设计

融合 Dieter Rams、原研哉与 Jony Ive 的设计智慧。Neo-Brutalist 风格，颜色服务于功能，而非仅仅装饰。

Principles

设计哲学

作为 Orange UI 整个设计系统的上位原则，而不是某个页面的装饰说明

Dieter Rams
"Weniger, aber besser"

少即是多，但要更好。好的设计是尽可能少的设计，专注于本质，去除一切不必要的元素。

原研哉 Kenya Hara
"白是一种追求，不是颜色"

空 (Emptiness) — 留白不是空无，而是充满可能性的容器。设计应该唤起感知，而非仅仅传递信息。

Jony Ive
"Inevitable design"

设计应该看起来"理所当然"，如同它本该如此。对精度的极致追求，通过微妙变化创造深度。

Foundations

系统基础

把上位原则转成颜色、层级、比例和交互的可执行规则

颜色语义

颜色先表达职责，再表达情绪。Orange UI 尽量减少色相数量，再用同色系的明暗与透明度建立层级。

primary

只用于主行动、激活态和系统主焦点。

accent

用于辅助强调、标签高亮和非危险性的视觉提点。

info / warning / success / error

蓝复用主操作色相，warning 与 accent 共享黄系，但通过深浅和对比度区分职责。

结构强度

Neo-Brutalist 的诚实不等于每个组件都同样强硬，而是让重要程度有清楚档位。

shadow-xs / sm

用于输入框、标签、小型控件和辅助容器。

shadow / lg

用于核心卡片、主按钮、反馈容器和浮层。

shadow-xl

只留给模态框等更高层级的叠加界面。

比例纪律

系统感来自可预测的比例，而不是组件数量。按钮、输入、分页和下拉共用同一套控制尺寸。

36 / 44 / 52

分别对应 small、default、large 控件高度。

2 / 4 / 8

圆角档位保持极少，避免每个组件都自成一套比例。

8pt spacing

继续作为模块与组件的主节奏，不随内容随意漂移。

交互语法

交互反馈应该诚实、可预期、低惊扰，优先表达状态变化，而不是卖弄动效。

hover

主控件统一轻微上浮，字段类组件则只强化边界与焦点。

active

统一使用轻微压下感，而不是缩放或夸张跳动。

focus / disabled

焦点清楚、禁用安静，永远不制造可点击的错觉。

什么是 Orange UI 的设计哲学？
Orange UI 融合了三位大师的设计智慧：Dieter Rams 的"少即是多"、原研哉的"白之美学"、以及 Jony Ive 的"理所当然的设计"。我们相信颜色应该服务于功能，而非仅仅装饰。
为什么选择 Neo-Brutalist 风格？
Neo-Brutalist 风格通过硬阴影和粗边框诚实地表达结构。正如 Dieter Rams 所说："好的设计是诚实的"——不夸大功能，不隐藏真实。
颜色系统的设计理念是什么？
我们的调色板使用温暖的奶油米白背景，搭配天蓝主色与柔黄辅助强调色。warning 不再额外引入新色相，而是在同一黄系里通过更深层级承担注意语义；信息继续复用主蓝。颜色服务于功能，也服务于整体视觉的舒适度。
